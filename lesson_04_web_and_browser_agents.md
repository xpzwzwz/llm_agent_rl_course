# 第四课：网页和浏览器 Agent

## 1. 为什么网页 Agent 重要

网页是最接近真实工作的 agent 环境之一。它包含：

- 搜索和阅读。
- 点击、输入、选择、提交。
- 多页面跳转。
- 动态内容和弹窗。
- 登录态和权限。
- 不完整或变化的信息。

网页 agent 能力强，通常说明模型已经具备一定环境观察、规划和恢复能力。

## 2. WebArena 的启发

WebArena 是一个真实网页风格的 agent benchmark，包含电商、论坛、内容管理、协作开发等环境。它的重要结论是：即使用强模型，真实网页任务成功率也远低于人类。

这说明网页 agent 的难点不只是“能不能点按钮”，而是：

- 读懂页面状态。
- 找到相关控件。
- 处理多步依赖。
- 在页面变化时调整策略。
- 判断最终任务是否完成。

## 3. browser-use 的工程启发

browser-use 是 GitHub 上很活跃的浏览器 agent 项目。根据 2026-05-21 的 GitHub API 查询，它约有 94,917 stars，最近 push 在 2026-05-20。

它代表了一种实用路线：

```text
LLM + 浏览器自动化 + 页面状态抽取 + 动作执行
```

训练数据可以从这类框架采集：

- 当前 URL。
- 页面摘要或可访问性树。
- 截图引用。
- 模型选择的动作。
- 动作执行结果。
- 最终任务状态。

## 4. 网页轨迹格式

网页轨迹可以这样记录：

```json
{
  "task": "在网站里找到订单 #123 的物流状态。",
  "steps": [
    {
      "url": "https://example.com/orders",
      "observation": "页面显示订单列表和搜索框。",
      "action": {
        "name": "type",
        "arguments": {"selector": "#search", "text": "123"}
      },
      "result": "搜索框已输入 123。"
    },
    {
      "action": {
        "name": "click",
        "arguments": {"selector": "button.search"}
      },
      "result": "页面显示订单 #123。"
    }
  ],
  "final_status": "success"
}
```

真实训练时，selector 不一定稳定。可以同时保存：

- DOM selector。
- 可访问性树节点。
- 元素文本。
- 元素相对位置。
- 截图或视觉特征。

## 5. Reward 怎么设计

网页任务 reward 可以分层：

- 最终成功：任务状态正确，例如订单状态被找到。
- 关键步骤：到达目标页面、提交正确表单。
- 安全约束：没有误删、误付款、误提交。
- 效率：步骤数不过长。
- 稳定性：没有重复点击、没有死循环。

不要只奖励“看起来像完成”。最好让环境提供可验证状态，例如数据库字段、页面文本、测试断言。
