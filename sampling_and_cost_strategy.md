# 采样和成本策略

## 1. 每个 prompt 采样几条？

起步：

```text
SFT 数据生成：1-3 条高质量
DPO 候选：4-8 条
GRPO：4-16 条
困难任务探索：16+ 条
```

采样数越高，成本越高，但更容易构造 success/failure pair。

## 2. pass@k 数据怎么用？

如果同一 prompt 采样 k 条：

- 成功样本可进 SFT 候选。
- 成功 vs 失败可构造 DPO。
- 多条成功中选成本更低、步骤更少的作为 chosen。
- 全失败任务进入错误分析或 curriculum 后期。

## 3. 高温采样是不是越多样越好？

不是。高温会增加多样性，也会增加格式错误。

建议：

- format 不稳时低温。
- 探索新策略时中高温。
- 采样后必须 verifier 过滤。

## 4. 如何判断数据边际收益下降？

看：

- 新数据带来的 success rate 增益。
- 新失败类型是否减少。
- 新样本与旧样本重复率。
- 训练成本和推理成本。
- 人工抽查通过率。

如果新增 10k 样本只带来模板重复，应停止扩量，转向补 hard cases。

## 5. 采样预算怎么分？

建议：

```text
已知高价值失败类型：高预算
新任务探索：中预算
已稳定简单任务：低预算
安全高风险任务：高审核预算
```

不要平均分预算。

## 6. 成本也能进 preference 吗？

可以。

当两条轨迹都成功：

```text
更少步骤
更少 token
更少昂贵工具调用
更少 wall time
```

可以作为 chosen。但不能用低成本压过正确性和安全性。
